# 社内アンケートシステム

社内向けの簡易アンケート（フォーム）収集ツールです。  
ローカル環境で動作し、管理者によるアンケート作成・集計と、一般ユーザーによる回答をサポートします。

---

## 機能一覧

| 機能 | 管理者 | 一般ユーザー |
|------|:------:|:-----------:|
| ログイン | ✓ | ✓ |
| アンケート一覧表示 | ✓ | ✓ |
| アンケート回答 | ✓ | ✓ |
| アンケート作成 | ✓ | — |
| 集計結果確認 | ✓ | — |
| ユーザー管理 | ✓ | — |

### 質問形式
- **自由入力** — テキストで自由に回答
- **プルダウン** — 選択肢から1つ選択
- **Yes / No** — はい・いいえの二択

---

## 技術スタック

- **バックエンド**: Python 3 / Flask
- **データベース**: SQLite
- **フロントエンド**: HTML / CSS / Vanilla JS（レスポンシブ非対応、PC専用）

---

## セットアップ

### 1. 依存パッケージのインストール

```bash
pip install -r requirements.txt
```

### 2. データベースの初期化

```bash
python init_db.py
```

初回実行時に `survey.db` が生成され、管理者ユーザーが登録されます。

### 3. アプリの起動

```bash
python app.py
```

ブラウザで http://127.0.0.1:5000 を開いてください。

## データベース構成

| テーブル | 概要 |
|----------|------|
| `users` | ユーザー情報（メール・パスワードハッシュ・ロール・表示名） |
| `surveys` | アンケート（タイトル・説明・期限・状態） |
| `questions` | 質問（質問文・形式・必須区分・表示順） |
| `question_options` | プルダウンの選択肢 |
| `answers` | 回答単位（誰がどのアンケートに回答したか） |
| `answer_details` | 質問ごとの回答内容 |

## ユーザー管理

管理者はヘッダーの「ユーザー管理」からユーザーを追加できます。  
パスワードは自動生成され、作成直後の画面にのみ表示されます。対象者への通知は手動で行ってください。

## 何を作って、何を諦めたか

### 実装したもの

| 項目 | 内容 |
|------|------|
| ログイン認証 | メールアドレス＋パスワードによるセッション管理 |
| ロールベースのアクセス制御 | 管理者／一般ユーザーで表示・操作を分離 |
| アンケート作成 | 自由入力・プルダウン・Yes/No の3形式、必須／任意設定 |
| アンケート回答 | 必須バリデーション、重複回答防止、回答済みの読み取り専用表示 |
| 集計結果 | 自由入力一覧・選択肢別人数・Yes/No比率グラフ・未回答者氏名一覧 |
| ユーザー管理 | 管理者による一般ユーザー追加、自動パスワード生成・画面表示 |

### 諦めたもの

| 項目 | 理由 |
|------|------|
| 回答期限アラート機能 | ヒアリングで「重要（必須ではない）」と位置付けられ、初期リリースの対象外とした。実現にはバッチ処理・メール送信サーバーが必要で、ローカル環境の構成要件を超えると判断した |
| レスポンシブ対応（モバイル） | クライアントより「PCのみで可」と明示されたため対象外とした |
| パスワードの自動メール通知 | ローカル環境にメールサーバーがないため、初期パスワードは管理者が画面上で確認して手動で通知する運用とした |
| アンケートの編集・削除 | ヒアリング・仕様書に要件として挙がらなかったため実装を見送った |
| 公開済みアンケートのステータス変更（締切操作） | 同上。DBのstatusカラムは設計上持っているが、UIからの操作は未実装 |

## 注意事項

- 本システムはローカル環境での利用を想定しています。インターネット公開は想定していません。
- `app.py` 起動時は Flask の開発サーバーを使用します。本番運用が必要な場合は Gunicorn 等の WSGI サーバーを利用してください。
